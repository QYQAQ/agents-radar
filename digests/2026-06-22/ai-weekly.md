## 2. OCR 与文档智能进展

**核心趋势：文档结构化向"推理驱动索引"迁移，传统向量检索范式受挑战**

| 项目/事件 | 关键进展 | 研究价值 |
|:---|:---|:---|
| **PaddleOCR** | 持续巩固 83K+ stars 标杆地位，"图像/PDF→结构化数据"定位深化 | LLM 文档理解 pipeline 的必备预处理组件，HMER 场景的基础视觉编码 |
| **VectifyAI/PageIndex** | "无向量、基于推理的 RAG"文档索引方案持续获关注 | 跳过敏密向量检索，直接以推理定位文档内容，对长文档层级化理解与版面分析有范式创新意义 |
| **yifanfeng97/Hyper-Extract** | 将非结构化文本转为超图/时空结构化知识 | 数学公式、科学文献的 HMER 后结构化新工具，单命令复杂抽取 |
| **RAGFlow** | 深度融合 RAG 与 Agent，"深度文档理解"模块含版面分析与表格识别 | 文档智能向交互式 Agent 演进，OCR 不再是终点而是推理起点 |
| **LEANN** | MLSys2026 工作，97% 存储压缩的本地 RAG | 大规模文档库内存高效索引，直接缓解长上下文场景下的文档检索成本 |

**关键观察**：OCR 层与 LLM 的边界正在模糊——PaddleOCR 等传统工具负责"视觉→文本"转换，而 PageIndex、Hyper-Extract 等新范式直接将文档内容纳入推理链条，跳过显式文本化阶段。这对 HMER 中"符号-视觉联合理解"的架构设计有启发。
